# IBM AI Builders Challenge Repository

## Mission

Build a polished, demonstrable entry for the 2026 AI Builders Challenge with IBM Bob.
The solution must target either the July theme, "Reimagine Creative Industries with AI,"
or the Wildcard theme, "Build Intelligent Systems for the Future of Work."

## Working agreements

- Read `ibm_ai_builders_challenge_brief.txt` before proposing features or architecture.
- Keep the selected challenge theme and target user explicit in product and technical decisions.
- Prefer a small, reliable end-to-end prototype over a broad collection of unfinished features.
- Treat accessibility, privacy, security, responsible AI, and human oversight as product requirements.
- Never commit API keys, credentials, private data, generated secrets, or local environment files.
- Keep setup reproducible and document every required dependency and environment variable.
- Add or update tests for important behavior, and run the relevant tests before declaring work complete.
- If no test command exists yet, state what was verified manually and add a test path when practical.

## Hackathon deliverables

- Maintain a public-repository-ready `README.md` with the problem, solution, architecture,
  selected theme, AI approach, setup steps, and a concrete account of how IBM Bob was used.
- Keep the prototype runnable for judges with minimal setup and useful sample data.
- Maintain a short demo path that can be shown clearly in a video of no more than three minutes.
- Record important IBM Bob contributions and significant human review or corrections.
- Use IBM Bob as the primary development tool. Other IBM AI technologies may supplement it.

## Decision priorities

When tradeoffs are necessary, optimize in this order:

1. A working, understandable prototype.
2. Clear challenge fit and real-world value.
3. Technical quality and responsible behavior.
4. Originality and memorable demonstration.
5. Additional features.
